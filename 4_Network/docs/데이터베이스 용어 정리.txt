데이터베이스(Database) / 스키마(Schema) -데이터를 저장하는 공간. MySQL에서는 둘을 같은 의미로 사용한다.

테이블(Table) -행(Row)과 열(Column)로 구성된 데이터 저장 단위.

컬럼(Column) -테이블의 속성, 데이터의 종류와 타입을 정의한다. 예: 카드ID, 카드명, 레벨, 설명, 카드 이미지 파일명

행(Row)  - 데이터 테이블 내부의 값.

기본키 Primary Key - 각 내용을 유일하게 지칭하는 값. PK가 아닌 키로 범위를 지정하면, 해당 내용이 포함된 모든 행을 대상으로 동작할 수 있기에 위험하므로, 특정 값만 수정하려면 PK를 활용해야한다.

복합키 Composite Key - 두 개 이상의 컬럼을 묶어서 기본키를 구성한 것.

트랜잭션 Transaction  -여러 SQL 작업을 하나의 “묶음 단위”로 처리하는 기능. DB 변경 및 수정의 최소 단위다.

START TRANSACTION -트랜잭션의 시작을 알리는 명령.

COMMIT -트랜잭션 내 모든 변경사항을 확정하고 DB에 반영.

ROLLBACK -트랜잭션 내 모든 변경사항을 취소하고 원래 상태로 되돌림.

원자성 Atomicity - 트랜잭션은 전부 적용되거나, 전혀 적용되지 않아야 함

일관성 Consistency - 트랜잭션 전후로 DB는 항상 의도된 단위에 맞게 유지되어야 함.
