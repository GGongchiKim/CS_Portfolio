

플랫폼(서비스): 사용자가 필요로 하는 특정 분야에 대한 여러 기능과 서비스를 모아둔 곳. ex) 네이버(인터넷 검색을 위한 플랫폼), 게임잡(게임관련 일자리와 관련된 서비스를 제공하는 플랫폼),  인벤(여러 게임 정보를 모아서 제공하는 플랫폼)

플랫폼(OS): 다양한 기기에서 사용되는 운영체제. ex) 안드로이드, IOS, 윈도우, PlayStation.

플랫폼(OS) 프로그래밍 : 서로 다른 환경을 가진 여러 운영체제에서 어떤 프로그램이 잘 작동할 수 있게 도구, 인증, 연결 등의 양식을 각 플랫폼이 요구하는 형태에 맞게 최신화, 최적화하는 작업.

Gradle -> 안드로이드에서 사용하는 개발도구의 일종
SDK -> SoftwareDevelopKit - 소프트웨어 개발에 필요한 도구들 (버전을 최신 플랫폼에 맞게 업데이트해줘야한다.)


두더지 잡기 Whack-a-Mole - 문제가 발생한 부분부터 수정해나가며, 이후 생기는 바깥쪽 문제들을 순서대로 해결하는 방식
엔진 스왑 Precise Renovation -원인이 되는 가장 중요한 부분부터 고치면서, 문제를 해결하는 방식 ( 주로 노후된 SDK나 라이브러리의 업데이트 등 현재 환경에서 문제가 될 수 있는 핵심요소부터 수정.)
빅뱅 접근법 The Real Big Bang -일단 다 최신버전으로 바꾸고 생기는 문제를 일일히 처리하기(가장 위험한 방식)
오류의 눈사태 -빅뱅 접근법처럼 처리하면 생기는 오류의 연쇄 발생




브레이킹 체인지 - 호환성이 상호계약 관계를 깨면서 진행되는 업데이트. 상호계약을 깨면서까지 해야 될만큼 중대한 변경이 있을때 이뤄짐. 주로 보안 문제나 구조 문제

*중요한 브레이킹 체인지*
AndroidX 라이브러리 도입(2018) -> support라이브러리 구조의 한계(유지보수, 버전충돌 등)를 극복하기 위해 도입됨.
런타임 권한 모델(2015) ->  앱을 설치할 떄 모든 권한에 한꺼번에 동의해야했던 것을 각각 변화함.
