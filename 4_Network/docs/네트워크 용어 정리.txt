네트워크 - 용어정리


TCP - 좀 더 줄 때와 받을 때 모두 체크하는 프로토콜. UDP에 비해 정합성을 확실히 따지지만 무거움.
UDP - 좀 더 가벼운 대신 받는 쪽이 제대로 받았나를 확인하지 않는 프로토콜.  

네트워크 프레임워크 - TCP와 UDP 사이에서 자신들의 프로그램에 필요한 수준의 보안을 챙기며, 최대한 가볍게 튜닝해둔 프로토콜.

DTO - 서버와 클라이언트가 데이터를 원활히 주고 받을 수 있게 규격화된 구조체 또는 객체. 서버와 클라이언트 서로의 수령함에 넣을 수 있게 디자인된 택배상자. 

JSON - 해쉬맵 , 딕셔너리를 텍스트 형태로 정리한 문서. key-value로 이어져 있고 실제로 파싱될 때 딕셔너리로 변환된다.


직렬화(Serialization) - 메모리 기반의 객체를 주소와 상관없이 재생성할 수 있는 구조로 변환시킨다. -> 이삿짐을 옮기기 위해 포장한다
역직렬화 - 직렬화된 구조를 사용할 수 있는 형태로 변환시키고 배치하는 과정. -> 도착한 이삿짐의 포장을 뜯고, 사용할 수 있게 물건들을 꺼내고 제대로 된 위치에 정리하는 과정.
ㄴ 파싱 - 박스를 까면서 쓸 수 있는 형태로 바꾸는 과정.
